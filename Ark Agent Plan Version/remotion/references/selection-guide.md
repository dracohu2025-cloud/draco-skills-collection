# Remotion Selection Guide

## 适合选 Remotion 的场景

优先选 Remotion，当你要的是：

- React 组件驱动的视频
- 模板化视频 / 批量生产视频
- 卡片、页面、UI、图表、仪表盘类内容
- 数据驱动的视频
- 需要复用前端样式系统、字体、布局、组件库

一句话：

> 如果你脑子里想的是“这一帧页面该长什么样”，Remotion 很可能对。

## 不适合优先选 Remotion 的场景

### 更适合 Manim
- 数学讲解
- 公式变换
- 几何推导
- 对象变换驱动的机制解释

### 更适合 Motion Canvas
- 场景式时间轴编排
- 偏 motion graphics 的 TS 动画
- 想一边写代码一边实时预览场景演出

## 快速判断题

### 选 Remotion，如果：
1. 你已经有 React / TS 代码或团队
2. 视频本质上像一个会动的网页 / 组件树
3. 未来可能做很多相似模板视频

### 别急着选 Remotion，如果：
1. 重点是数学对象本身怎么变
2. 重点是镜头编排感，而不是组件复用
3. 只是因为“我会 React”，但视频其实是另一种问题

## 简版决策

- **对象变换讲解** → Manim
- **React 组件视频** → Remotion
- **场景编排动画** → Motion Canvas
