# Decision Table

| 情况 | 更像哪个框架 | 原因 |
|---|---|---|
| 数学/公式/几何/算法推导 | Manim | 对象变换表达力最强 |
| 技术讲解动画、场景编排、TS 动画 | Motion Canvas | 场景和时间轴更顺手 |
| React 页面视频、卡片、模板、批量生成 | Remotion | 组件复用和工程化最强 |
| 主要问题是“这一帧页面长什么样” | Remotion | 天然按组件/页面建模 |
| 主要问题是“这个对象怎么变化” | Manim | 天然按对象/变换建模 |
| 主要问题是“这个场景怎么演” | Motion Canvas | 天然按场景/时间轴建模 |

## 最小判断法

先问一句：

- 你更像在拍 **对象**、**场景**、还是 **页面**？

答案通常已经够用了：

- 对象 → Manim
- 场景 → Motion Canvas
- 页面 → Remotion
